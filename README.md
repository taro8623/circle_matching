現在のタスクの状況と次の一手
1. JWTのログインフローの完成確認
JWTログインフローの完成確認  
ここは「発行 → 署名 → 返却 → クライアント保持 → 認証付きリクエストで送信」の一連が動くかをチェックする段階だね。
特に確認したいのは：
アクセストークンの有効期限
署名アルゴリズムの統一（バックエンドとフロントで一致してるか）
エラーハンドリングの統一（401/403 の扱い）
2. 保護されたAPIへの認証付きアクセスの実装
保護APIへの認証アクセス  
ここは JWT を Authorization: Bearer <token> で送って、
サーバ側でミドルウェアが検証 → コントローラへ、という流れを作る部分。
チェックポイントは：
ミドルウェアでのトークン検証
ロール/権限の判定（必要なら）
期限切れトークンの扱い
3. リフレッシュトークン導入の検討
リフレッシュトークン導入  
ここは設計方針によって大きく変わるから、決めるべきポイントが多い。
保存場所（HttpOnly Cookie / DB / Redis）
ローテーション方式（毎回更新するか、固定にするか）
セキュリティ要件（盗難時の無効化戦略）
4. チーム開発を見据えたコード構造の整備
チーム開発向けコード構造  
ここは後回しにすると地獄を見るやつなので、今のうちにやるのは正解。
ディレクトリ構造の標準化
責務ごとの分離（controller / service / repository）
依存関係の明確化（DIコンテナ使うかどうか）
環境変数管理の統一（dotenv / secrets manager）
